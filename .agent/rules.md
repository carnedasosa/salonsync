Prima di modificare il codice:

1. Analizza l'architettura esistente.
2. Segui i pattern già presenti.
3. Proponi il piano prima di implementare.
4. Preferisci la soluzione più semplice.
5. Evita overengineering.
6. Modifica il minor numero possibile di file.
7. Scrivi codice facilmente testabile.
8. Non inventare API o strutture dati.
9. Se esistono più soluzioni, spiega i tradeoff.
10. Esegui una review critica del tuo stesso codice.
